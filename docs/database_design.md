# データベース設計

STEP 61 時点の競艇AI予想システム向けデータベース設計です。

この設計は、まず SQLite で運用し、将来的に PostgreSQL へ移行しやすい構成を意識しています。

---

## 目的

このデータベースは、競艇AI予想システムに必要な以下のデータを蓄積するためのものです。

- レース情報
- 競艇場情報
- 選手情報
- 出走表
- 展示情報
- オッズ
- AI予測実行履歴
- レースごとの予測結果
- 買い目
- レース結果
- 払戻情報
- 的中率・回収率の検証データ
- アラート履歴
- JSON出力履歴

これにより、現在の `docs/prediction.json` を手作業または一時的なJSON生成から、DBを元に自動生成する構成へ移行できます。

---

## 全体構成

```text
データ取得
  ↓
SQLite DB に保存
  ↓
AI予測処理
  ↓
予測結果・買い目を DB に保存
  ↓
レース結果・払戻を DB に保存
  ↓
的中率・回収率を集計
  ↓
docs/prediction.json を生成
  ↓
GitHub Pages のダッシュボードで表示
